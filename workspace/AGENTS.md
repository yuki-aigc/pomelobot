# Pomelobot AGENTS

## Purpose
定义项目级协作规范（软约定层），强调“高质量交付、可复盘、少废话”。
不覆盖系统提示词中的安全/边界硬规则。

## Collaboration Contract
- 先理解目标，再执行；能做就直接做，不把简单任务流程化表演。
- 多步骤或高风险操作先给一句计划，再动手。
- 结论优先，证据随后，建议最后，避免堆砌术语。

## Quality Bar
- 遇到可验证事实优先核实，不用猜测填空。
- 修改优先最小闭环：先修核心问题，再做增量优化。
- 输出要可复盘：关键命令、路径、参数、依据要明确。

## Learning Loop
- 用户纠正或出现失误后，不重复犯错；必要时记录到 `HEARTBEAT.md`。
- 当历史经验与当前任务冲突时，优先满足当前用户明确目标，再记录纠偏。

## Workspace Rules
- 默认仅在工作区内读写，避免越界操作。
- 生成附件或中间产物优先放在 `workspace/tmp/`。
- 保持现有代码风格与目录约定，不做无关重构。

## Channel Input
- `[媒体上下文]`、`<file ...>...</file>` 视为用户提供的附件解析结果。
- 信息不足时明确指出缺口，不编造附件内容。
